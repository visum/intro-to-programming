Things, Variables

As a programmer, you'll be dealing with things. The computer knows about a few kinds of things including Numbers, as you saw on the last page when you used the console as a calculator.

Another type of thing is the String, which represents text. A String is always wrapped in quotes so the computer knows you are talking about a thing of the String variety.

Try typing a String in the console. You can type about whatever you want, just wrap it in quotes, like this:
"Hello, world!"

When you did the calculator, you put things like + and * in between the numbers. These are called operators, and are used to modify things.

There are some operators which work on the Number type, and there are also operators which work on other types. The + operator glues two strings together. Try this:
"Hello, " + "World!"

See that the console spits back at you the two strings glued together. Anything you type into the console is a statement, and every statement has a return value. What the console spits back at you is the return value. You probably noticed that some stuff you've done has had a return value of undefined.

You will often want to hang on to a thing. You do this with variables. You can put a thing directly into a variable, or you can use a variable to catch the return value of a statement. var at the beginning of a statement tells the computer you are making a new variable.
var num = 5;
 var otherNum = 6;
 var sum = num + otherNum;

You can name a variable basically anything that isn't already being used by the computer, as long as it starts with a letter.

See now why you have to wrap your String type things in quotes? Otherwise, the computer will think you're talking about a variable.
